name: dig-and-advise-pattern
description: User prefers "dig and advise" workflow — research tradeoffs, present recommendation with rationale, let user accept/reject
type: feedback
---

When multiple settings or changes need consideration, create task lists and work each independently. Don't batch decisions.

**Why:** The user wants to weigh each item on its merits, not rubber-stamp a bundle. "Dig and advise" means: research first, present a recommendation with tradeoffs, wait for approval before implementing.

**How to apply:** For any non-trivial configuration or design decision, present the analysis and a clear verdict (implement / skip / defer) with rationale. "Approved for obvious" means implement the clear wins and hold back on judgment calls.
