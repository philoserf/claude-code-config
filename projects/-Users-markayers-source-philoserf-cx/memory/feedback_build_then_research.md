description: User prefers learning by building, then checking prior art — not the other way around

User explicitly asked to research prior art only after the full implementation was complete. Building first provides hands-on understanding of the problem space before evaluating alternatives.

**Why:** Building reveals constraints (like JXA performance) that reading about wouldn't. The user values direct experience.

**How to apply:** Don't front-load prior art research before implementation unless the user asks. Offer it as a post-implementation step.
