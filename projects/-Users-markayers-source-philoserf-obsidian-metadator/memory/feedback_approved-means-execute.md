When Mark reviews a proposed approach and says "approved" (or equivalent), that means execute now. Don't generate a formal plan document or ask for execution strategy — just start working.

**Why:** Mark's preference is "action over explanation." Writing a plan for well-defined issues with clear descriptions is busywork.

**How to apply:** If issues are already well-defined (e.g., GitHub issues with descriptions), skip the plan document and go straight to implementation after approval.
