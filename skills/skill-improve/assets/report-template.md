# Report Template

This document defines the standardized output format for improvement reports.

## Report Structure

```markdown
# Improvement Report: {skill-name}

**Evaluated**: {YYYY-MM-DD}
**Location**: {path/to/skill}

## Summary

{1-2 sentence overview of skill and improvement opportunities}

**Improvement potential**: {High/Medium/Low}
**Estimated total effort**: {X hours for all recommendations}

## Recommendations

### P1: Do First

{High impact, low effort - do these immediately}

#### 1. {Recommendation title}

**Category**: {Effectiveness/Clarity/Best Practices/Documentation/Verification/Trigger Coverage}
**Effort**: ~{X} minutes
**Impact**: {What improves}

**Current**:
{What exists now}

**Recommended**:
{What it should be}

**Why**: {Justification}

---

#### 2. {Recommendation title}

...

---

### P2: Plan Carefully

{High impact, high effort - worth the investment}

#### 3. {Recommendation title}

**Category**: {Effectiveness/Clarity/Best Practices/Documentation/Verification/Trigger Coverage}
**Effort**: ~{X} hours
**Impact**: {What improves}

**Current**:
{Description of current state}

**Recommended**:
{Description of target state}

**Implementation notes**:
- {Step 1}
- {Step 2}
- {Step 3}

**Why**: {Justification}

---

### P3: Quick Wins

{Medium impact, low effort - batch these together}

#### 4. {Recommendation title}

**Category**: {Effectiveness/Clarity/Best Practices/Documentation/Verification/Trigger Coverage}
**Effort**: ~{X} minutes
**Impact**: {What improves}

**Change**: {Brief description of change}

---

{Medium impact, high effort - weigh cost vs benefit}

#### 5. {Recommendation title}

**Category**: {Effectiveness/Clarity/Best Practices/Documentation/Verification/Trigger Coverage}
**Effort**: ~{X} hours
**Impact**: {What improves}

{Implementation guidance}

---

### P5: Nice to Have

{Low impact - optional polish}

- {Item 1}: {brief description}
```

## Report Guidelines

### Specificity

- Point to exact lines, files, or text when possible
- Show before/after for clarity
- Include specific wording for trigger phrases

### Actionability

- Each recommendation should be immediately implementable
- Include concrete examples, not just concepts
- Provide actual text/code when helpful

### Prioritization

- Always assign a priority level
- Group by priority in the report
- Include effort estimates

### Justification

- Explain why each improvement matters
- Connect to user experience or skill quality
- Help reader make informed decisions about P4 items

## Abbreviated Report Format

For quick assessments:

```markdown
# Quick Improvements: {skill-name}

## Do First (P1)
1. **{title}** - {one-line description}
2. **{title}** - {one-line description}

## Plan Carefully (P2)
3. **{title}** - {one-line description}

## Quick Wins (P3)
4. **{title}** - {one-line description}

**Total estimated time**: {X hours}
```

## Category-Focused Report Format

When user requests improvements in specific category:

```markdown
# {Category} Improvements: {skill-name}

## Current State

{Assessment of current state for this category}

## Recommendations

### 1. {Recommendation} (P{X})

**Current**:
{what exists}

**Recommended**:
{what should be}

### 2. {Recommendation} (P{X})

...

## Summary

- **P1 items**: {count}
- **P2 items**: {count}
- **Total effort**: {X hours}
```

## Comparison Report Format

When comparing improvement needs across skills:

```markdown
# Improvement Comparison

| Skill | P1 Count | P2 Count | Total Effort | Priority |
|-------|----------|----------|--------------|----------|
| {skill 1} | {n} | {n} | {X hrs} | {High/Med/Low} |
| {skill 2} | {n} | {n} | {X hrs} | {High/Med/Low} |

## Recommended Focus Order

1. **{skill}** - {reason for priority}
2. **{skill}** - {reason}

## Common Issues Across Skills

- {Issue appearing in multiple skills}
- {Issue appearing in multiple skills}
```

## Quality Indicators for Reports

A good improvement report:

- [ ] Has at least one P1 or P2 recommendation (or explains why none exist)
- [ ] Provides specific, actionable guidance
- [ ] Includes effort estimates
- [ ] Shows before/after for complex changes
- [ ] Groups recommendations by priority
- [ ] Includes implementation order suggestion
