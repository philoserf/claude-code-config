## Report Guidelines

### Completeness

- Fill in all sections (use "N/A" if truly not applicable)
- Provide at least one strength and one issue per dimension
- Include specific line numbers or file references where helpful

### Objectivity

- Base scores strictly on rubric criteria
- Cite specific evidence for each score
- Avoid subjective language ("I think", "seems like")

### Actionability

- Make issues specific enough to fix
- Prioritize issues by impact (implicit in dimension weights)
- Keep summary focused on most important points

### Formatting

- Use consistent markdown formatting
- Include calculation table for transparency
- Keep individual sections concise

## Abbreviated Report Format

For quick assessments or comparisons, use this shorter format:

```markdown
# Quick Assessment: {skill-name}

**Score**: {X.X}/5.0 ({tier})

| Dimension | Score |
|-----------|-------|
| Effectiveness | {1-5} |
| Clarity | {1-5} |
| Verification | {1-5} |
| Documentation | {1-5} |
| Best Practices | {1-5} |
| Trigger Coverage | {1-5} |

**Top strength**: {one sentence}
**Top improvement**: {one sentence}
```

## Comparison Report Format

When comparing multiple skills:

```markdown
# Quality Comparison

| Skill | Score | Tier | Strongest | Weakest |
|-------|-------|------|-----------|---------|
| {skill 1} | {X.X} | {tier} | {dimension} | {dimension} |
| {skill 2} | {X.X} | {tier} | {dimension} | {dimension} |
| {skill 3} | {X.X} | {tier} | {dimension} | {dimension} |

## Detailed Comparison

### {Dimension with largest variance}
{Analysis of why skills differ on this dimension}

### Recommendations
- {skill 1}: {key recommendation}
- {skill 2}: {key recommendation}
```
