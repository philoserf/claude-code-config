# Report Template

Standardized output format for quality assessment reports.

## Full Report

```markdown
# Quality Assessment: {skill-name}

**Evaluated**: {YYYY-MM-DD}
**Location**: {path/to/skill}

## Overall Score

| Metric             | Value       |
| ------------------ | ----------- |
| **Weighted Score** | {X.XX}/5.00 |
| **Quality Tier**   | {tier}      |

## Dimension Scores

| Dimension        | Weight | Score | Contribution   |
| ---------------- | ------ | ----- | -------------- |
| Effectiveness    | 28%    | {1-5} | {score × 0.28} |
| Clarity          | 22%    | {1-5} | {score × 0.22} |
| Best Practices   | 17%    | {1-5} | {score × 0.17} |
| Documentation    | 15%    | {1-5} | {score × 0.15} |
| Verification     | 10%    | {1-5} | {score × 0.10} |
| Trigger Coverage | 8%     | {1-5} | {score × 0.08} |
| **Total**        | 100%   |       | **{sum}**      |

## Dimension Analysis

### Effectiveness ({score}/5)

{Evidence and rationale}

### Clarity ({score}/5)

{Evidence and rationale}

### Best Practices ({score}/5)

{Evidence and rationale}

**Metrics**:
- SKILL.md: {lines} lines, {words} words
- Reference files: {count}
- Frontmatter fields: {list documented/undocumented}

### Documentation ({score}/5)

{Evidence and rationale}

**Files reviewed**:
- SKILL.md ({line count} lines)
- {reference files with line counts}

### Verification ({score}/5)

{Evidence and rationale. Note skill type: task/analysis/reference.}

### Trigger Coverage ({score}/5)

{Evidence and rationale}

**Description**: "{full description text}"

## Summary

**Strengths**:
1. {key strength}
2. {key strength}

**Improvements** (highest impact first):
1. {actionable fix}
2. {actionable fix}

---
*Assessment generated by skill-quality*
```

## Guidelines

### Keep it evidence-based

- Cite specific line numbers, file paths, or quoted text
- Every score needs at least one concrete observation
- Avoid subjective language ("I think", "seems like")

### Keep it actionable

- Improvements should be specific enough to implement
- Order improvements by impact (dimension weight x score delta)
- Skip "Improvements" section entirely if all dimensions score 5

### Keep it proportional

- Dimensions scoring 5: evidence only, no improvements needed
- Dimensions scoring 3-4: brief evidence + specific suggestion
- Dimensions scoring 1-2: detailed evidence + priority fix

### Abbreviated format

For quick assessments or batch comparisons:

```markdown
**{skill-name}**: {X.XX}/5.00 ({tier})
  Eff={n} Clar={n} BP={n} Doc={n} Ver={n} Trig={n}
  Top strength: {one line}
  Top fix: {one line}
```

### Comparison format

When comparing multiple skills:

```markdown
# Quality Comparison

| Skill       | Score | Tier   | Strongest        | Weakest          |
| ----------- | ----- | ------ | ---------------- | ---------------- |
| {skill 1}   | {X.X} | {tier} | {dimension}      | {dimension}      |
| {skill 2}   | {X.X} | {tier} | {dimension}      | {dimension}      |

**Largest variance**: {dimension} — {analysis of why skills differ}
```
